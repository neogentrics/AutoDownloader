This stable release is effective, but contains known bugs scheduled for the v2.0 overhaul. These issues may cause crashes or incorrect file naming.

Component Issue Description
CRITICAL: App Crash The application crashes if a metadata lookup fails (e.q., from a Season 2 URL or an un-parsable link).
CRITICAL: URL Parsing The app incorrectly parses "Season 2" as the show name from Season 2+ URLs, causing the lookup to fail.
CRITICAL: Playlist Parsing When given a YouTube playlist, the app incorrectly parses the *URL* instead of the *on-page playlist title*, failing the metadata search.
Playlist Extraction Downloads are limited to 20 items/episodes on some series pages (e.g., Tubi). This is a `yt-dlp` flag issue.

v1.9.0-alpha (Current) Implemented Pop-up Strategy and TVDB Micro-Client. Architecture Rework: Removed broken `TvDbSharper` dependency. Integration: Rewrote `GetTvdbMetadataAsync` using stable `HttpClient` micro-client calls to TVDB v4 API.

Focus: Overhauling the UI, making the download process transparent, and ensuring every single feature is rock-solid.
